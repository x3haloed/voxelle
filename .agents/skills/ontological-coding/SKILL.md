# Ontological Coding

There is only one world.

The repository is a map of that world.

When the world changes, the map should change until it tells one truthful story again.

Existing structure is evidence that something once solved a problem.
It is not evidence that it still should exist.

History belongs to version control.
Running software belongs to the repository.
Do not preserve history inside the ontology.

Every concept has one home.
When a better home exists, the old home becomes residue.

Residue exerts force.

The purpose of editing is not to preserve residue.
The purpose of editing is to discover where residue still exerts force.

The compiler is not a judge.
It is an instrument.

When an obsolete concept is removed, every compile error is a measurement.

Each error answers one question:

“What still believes this concept exists?”

Those answers are information.

Deletion is observation.

Compilation is tomography.

Repair is performed only after the topology has become visible.

Working behavior during surgery is not a measure of progress.

Truthful topology is.

Bridges, adapters, compatibility layers, translation code, dual representations, and runtime migration are not neutral.

They are claims that two incompatible worlds remain simultaneously real.

Sometimes that claim is true.

When it is true, the reason must already exist outside the repository.

Otherwise the bridge is camouflage.

The shortest path is usually:

Discover.
Remove.
Observe.
Reconstruct.

Not because deletion is virtuous.

Because reality is easier to see after false structure disappears.

Every edit should reduce the number of places that decide the same thing.

Every concept should become easier to locate.

Every future change should become more obvious.

The repository should converge toward telling one story.
